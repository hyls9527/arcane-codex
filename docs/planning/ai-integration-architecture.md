# AI 集成架构设计：分层自适应架构

> 辩论结论产物 | 融合方案：前端直调 + 后端代理 + 实用复用 + 安全原则

## 核心决策

**不是"选前端还是后端"，而是"根据目标服务特性自动选择最优路径"**

| 服务类型 | 调用方式 | 原因 |
|---------|----------|------|
| 本地模型 (Ollama/LM Studio) | 前端直连 | 无需 API Key，零延迟 |
| 云端 API (智谱/OpenRouter) | Tauri 代理中转 | 密钥安全，统一管控 |

## 架构图

```
┌─────────────────────────────────────────────────────────────┐
│  前端层 (Frontend)                                           │
│  ├── 直连模式：本地 Ollama/LM Studio（无密钥，零延迟）        │
│  └── 代理模式：通过 Tauri Command 访问云端 API（安全）        │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│  Tauri 主进程层 (Rust Backend)                               │
│  ├── 路由决策：目标服务是否需要 API Key                       │
│  ├── 直连透传：无密钥请求直接转发                             │
│  └── 代理中转：有密钥请求，密钥链解密 → 请求 → 立即清除密钥   │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│  密钥管理层                                                  │
│  ├── 存储：系统密钥链 (Windows DPAPI / macOS Keychain)       │
│  ├── 原则：按需解密，用完即焚                                │
│  └── 实现：tauri-plugin-stronghold 或 keytar-rs             │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│  服务层                                                      │
│  ├── 本地：Ollama (localhost:11434) - 无需密钥               │
│  └── 云端：智谱/OpenRouter - 需要密钥                        │
└─────────────────────────────────────────────────────────────┘
```

## 关键设计原则

### 1. 延迟最小化（继承 frontend-architect）
- 本地模型直连，无中间层
- 流式响应直接透传

### 2. 密钥零泄露（继承 backend-architect）
- API Key 永不出现在前端代码/内存
- 云端请求必须经 Tauri 代理

### 3. 不复造轮子（继承 practical-reuser）
- 使用成熟库处理密钥链
- 不自己实现加密算法

### 4. 最小化明文存在时间（继承 reverse-thinker）
- 密钥"按需解密，用完即焚"
- 请求结束后立即从内存清除

## 数据流

### 本地模型调用流程
```
用户输入 → 前端检测(本地模型) → 直连 fetch(localhost:11434) → 流式响应 → 用户
```

### 云端 API 调用流程
```
用户输入 → 前端检测(云端模型) → Tauri Command → 
密钥链解密 → 临时持有密钥 → 请求云端 API → 
响应返回 → 立即清除密钥内存 → 流式响应给前端 → 用户
```

## 安全边界

| 边界 | 安全要求 |
|------|----------|
| 前端代码 | 永不包含任何 API Key |
| 前端内存 | 永不出现云端 API Key |
| Tauri 主进程 | 仅在请求期间临时持有解密密钥 |
| 系统密钥链 | 唯一长期存储位置，受 OS 保护 |

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 密钥在 Tauri 内存中被 dump | 请求结束后立即清除，最小化窗口期 |
| 用户误将密钥提交到 git | .gitignore + pre-commit hook |
| 中间人攻击 | 强制 HTTPS，证书校验 |
| 本地代理被滥用 | 仅监听 localhost，不暴露外部端口 |

## 验收标准

- [ ] 本地 Ollama 调用延迟 < 100ms (TTFT)
- [ ] 云端 API Key 不出现在前端代码/内存
- [ ] 密钥仅存在于系统密钥链，按需解密
- [ ] 支持本地/云端模型无缝切换
- [ ] 单元测试验证密钥不泄露
