# 一个18岁穷大学生的小米Token申请自述

## 我叫鳕鱼，今年18岁

我每个月的生活费就那么多。除去吃饭的钱，剩下的全部砸进了买token里。

不是因为我想炫富，而是因为我发现了一件事——**AI是我这辈子见过的最好的创造工具**。

但用AI很贵。一个项目跑下来，几百上千的token说没就没。我咬着牙从生活费里抠，省一顿饭换一个prompt的试错。

然后，我遇到了小米的OpenClaw，第一次用上了免费的token。

就是那一次，我脑子里的点子开始疯长。

## 第一件成就：Arcane Codex —— 让照片变成记忆

我拍了很多照片。几万张，躺在硬盘里。找一张照片的感觉，就像在垃圾堆里翻宝贝——明明知道它在那，就是找不到。

市面上的照片管理工具，要么太简单（系统自带的），要么太复杂（企业级DAM），要么把你的照片传到别人的服务器上。

我不想妥协。所以我自己写了一个。

**Arcane Codex。** 用React + TypeScript做前端，Rust + Tauri 2.0做后端，SQLite做数据库。AI理解照片内容，语义搜索找到你想要的那一张。全部本地运行，数据永远属于你。

### 我遇到了什么问题？

架构设计的时候，我和自己吵了一架。

前端想直接调本地模型——延迟低，体验好。但云端API的Key放哪？前端？太危险了。

后端想全走代理——安全是安全了，本地模型白白多过一层，延迟翻倍。

两个都有道理。谁也说服不了谁。

### 我是怎么解决的？

我没有选任何一边。我发明了第三条路——**分层自适应架构**。

本地模型（Ollama/LM Studio）前端直连，不需要Key，零延迟。云端API（智谱/OpenRouter）走Tauri代理中转，密钥存在系统Keychain里，按需解密，用完即焚。

不是前端vs后端，而是**根据目标服务特性自动选择最优路径**。

这个方案的产物文档上写着："辩论结论产物 | 融合方案"。对，我管它叫辩论——我左边站一个frontend-architect，右边站一个backend-architect，我自己当裁判。最后两边都赢了。

202个开发任务，测试覆盖率96.7%。全部搞定。

## 第二件成就：Seedance AI导演系统 —— 用24个行业的语言跟AI对话

用小米免费OpenClaw的那段时间，我脑子里突然冒出一个想法：

**能不能用专业的电影术语，让AI帮我生成视频提示词？**

不是那种"一个好看的女孩在海边"的废话，而是真正的导演级别——景别、运镜、光影、叙事语法、180°法则、希区柯克变焦。

于是我做了两件事：

1. 整理了**24个专业行业的术语库**——影视制作、摄影灯光、动画动漫、视觉特效、游戏美术、建筑设计、时尚美妆、广告营销、医疗、航空、法律、金融、工业……每个行业的术语精确到能被导演拿来用。

2. 写了一套**Seedance Storyboard系统**——用户粘贴剧本，自动拆解为分镜提示词，直出可复制到即梦/Seedance 2.0的完整提示词。五要素导演法、情绪-运镜映射表、首帧参考一致性、台词时长计算公式（字数÷3.5）。

这中间最大的坑是什么？

**AI不懂什么叫"一句话以内"。** 我让它用一句话描述分镜，它给我写三行。我让它控制节奏，它把高潮放在第三秒。我让它做转场逻辑，它一路硬切到底。

怎么解决的？

我写了**一套导演脚本→提示词的转化引擎**。不是让AI自由发挥，而是先把剧本分析、场景切割、角色卡、情绪转折点全部算清楚，然后按模板转化。AI只是执行者，规则我来定。

那个seedance-pipeline-v72里，241个文件，音频分析样本、场景分类脚本、提示词验证脚本、策略指南、质量评分表……全是我一遍遍试出来的。

`crypto-arb-cn`、`polymarket-auto-trader`、`elliott-wave-principle`……删了。全部删了。

不是因为它们不好，而是因为我发现了一个更重要的东西：**专注比什么都强。**

我现在保留的77个Skills，每一个都是我真实用过的：
- mimo-omni：多模态理解（图/视频/音频）
- multi-search-engine：17个搜索引擎聚合

甚至我自己写了两个Role：
- **欢愉令使·阿哈的使者**：不是写笑话，是编织不协调——建立预期，精准打破，但刚好在"安全又好笑"的区间。
- **押韵星神·莱姆**：让歌词赖在脑子里不走。笑是入门，忘不了才是本事。

## 第四件成就：Ralph Engine —— 让AI不只是聊天，而是真正做事

我写了一个Python执行引擎。

意图层 → 翻译层 → 调度层 → 执行层 → 验证层 → 反馈层。

状态机：IDLE → PLANNING → DISPATCHED → EXECUTING → VERIFYING → COMPLETED/FAILED。

这不是玩具。这是让AI从"说得好听"变成"做得到位"的基础设施。
