# 项目需求文档 (PRD)

## 1. 项目概述

### 1.1 背景
原项目 `arcane-codex-src` (位于 `D:\Personal\Desktop\arcane-codex-src\`) 是一个基于 Flask + Python 的本地图片知识库原型，存在进程管理复杂、安装包膨胀 (~200MB)、AI 批量处理无断点续传等架构缺陷。本次为全新重建，采用纯 Rust Tauri 2.x 架构，实现轻量级、高性能、本地优先的图片知识管理系统。

### 1.2 目标
构建一款基于 **Tauri 2.x (纯 Rust) + React 18 + SQLite + LM Studio** 的桌面应用，通过多模态 AI 实现智能图像理解、隐私安全、离线可用、极具创新视觉体验的个人知识管理系统，支持管理 5 万+ 张图片。

### 1.3 目标用户
- **个人收藏者**: 整理旅行照片、设计素材、截图收藏
- **内容创作者**: 管理设计灵感、参考图库、mood board
- **隐私敏感用户**: 本地化存储、完全离线操作、拒绝云端上传
- **AI 研究者**: 测试多模态模型、批量处理图像数据

### 1.4 核心设计哲学
| 原则 | 说明 |
|------|------|
| **Local-First** | 所有数据存储、AI 推理均在本地完成 |
| **Privacy-First** | 不上传任何用户数据，不采集遥测信息 |
| **Zero-Config** | SQLite 嵌入式数据库，开箱即用 |
| **Modular** | 清晰分层架构，支持未来插件扩展 |
| **Progressive** | MVP 先保证基础功能，后续逐步增强 |

## 2. 详细功能设计 (Detailed Design)

### 2.1 模块 A: 图片导入与管理 (Image Management)

#### 2.1.1 页面: 主图库 (Gallery)
**路由**: `/` (默认首页)

**UI 元素清单**:
| 元素名称 | 类型 | 验证规则 | 默认值 | 交互逻辑 | 异常处理 | 权限 |
|---|---|---|---|---|---|---|
| 图片网格 | Virtual Grid | 仅渲染可视区域 | 200px 缩略图 | 滚动懒加载，60fps | 显示占位符 | 所有用户 |
| 拖拽区域 | DropZone | 支持 JPEG/PNG/WebP/HEIC/GIF | 无 | 拖入文件/文件夹触发导入 | 提示不支持格式 | 所有用户 |
| 搜索框 | Input | 支持关键词搜索 | 空 | 输入即搜索，debounce 300ms | 无结果友好提示 | 所有用户 |
| 筛选器 | Select | 按状态/分类/日期 | 全部 | 组合筛选，实时更新 | - | 所有用户 |
| 状态面板 | Status Bar | 显示总数/AI 进度 | 实时 | 点击展开详情 | - | 所有用户 |
| 主题切换 | Toggle | Light/Dark/System | System | 一键切换，300ms 过渡 | - | 所有用户 |

**逻辑流程**:
1. 用户打开应用 → 加载本地 SQLite 数据库
2. 渲染图片网格 (react-virtual 虚拟滚动)
3. 用户拖拽图片 → Rust 端验证文件 → 生成缩略图 → 写入数据库
4. 前端实时更新，显示新导入图片

**图片存储策略**:
- **默认模式**: 索引引用 (不移动原文件，仅记录文件路径和元数据到数据库)
- **可选增强**: "归档到知识库"按钮 → 复制图片到应用数据目录 (`%APPDATA%\ArcaneCodex\images\`)
- **断链检测**: 定期扫描索引图片是否被删除/移动，标记失效记录 (状态: `broken_link`)
- **安全导出**: 将索引图片批量复制到用户指定位置，便于备份/分享

**缩略图存储策略**:
- **存储位置**: `%APPDATA%\ArcaneCodex\thumbnails\{image_id}.webp` (image_id 为数据库主键)
- **生成规则**: 最大尺寸 300x200px，保持宽高比，WebP 格式 quality=80
- **清理策略**: 删除图片记录时同步删除对应缩略图文件 (Rust 端 `std::fs::remove_file`)
- **缓存失效**: 原文件修改后 (通过 file_hash 变更检测) 重新生成缩略图

#### 2.1.2 页面: 图片详情 (ImageViewer)
**路由**: `/image/:id`

**UI 元素清单**:
| 元素名称 | 类型 | 交互逻辑 |
|---|---|---|
| 大图预览 | Image | 支持缩放 (滚轮)、平移 (拖拽)、旋转 |
| 元数据面板 | Panel | 显示 EXIF 信息、文件大小、尺寸 |
| AI 标签列表 | Tags | 显示 5-10 个 AI 生成标签，可点击筛选 |

| 操作按钮组 | Toolbar | 删除/归档/重新分析/导出 |

**逻辑流程**:
1. 点击网格中的图片 → 导航到详情页
2. 加载原图 (非缩略图) + AI 分析结果
3. 用户可缩放/平移查看细节
4. 可编辑/补充标签 (手动打标)

### 2.2 模块 B: AI 自动打标 (AI Auto Tagging)

#### 2.2.1 页面: AI 处理面板 (AIProgressPanel)
**入口**: 侧边栏 → "AI 打标"

**UI 元素清单**:
| 元素名称 | 类型 | 交互逻辑 | 异常处理 |
|---|---|---|---|
| 进度条 | ProgressBar | 实时显示完成百分比 | - |
| 统计计数 | Stats | 成功/失败/重试计数 | - |
| 暂停按钮 | Button | 暂停 AI 队列 | 状态变为"已暂停" |
| 继续按钮 | Button | 恢复 AI 队列 | 状态变为"处理中" |
| 取消按钮 | Button | 取消所有待处理任务 | 二次确认弹窗 |
| 结果列表 | List | 显示最近处理结果 | 失败项红色标记 |
| 重试按钮 | Button (Per Item) | 重新提交失败任务 | 加入队列末尾 |

**逻辑流程**:
1. 导入图片后自动创建 `task_queue` 记录 (status = pending)
2. Rust 后台 Worker 扫描队列，调用 LM Studio API
3. 每个 Worker 获取 Semaphore permit (默认并发 3)
4. AI 分析完成 → 更新数据库 → emit 事件 → 前端刷新
5. 失败 → retry_count++，最多 3 次，指数退避

**LM Studio 连接管理**:
- **启动检测**: 应用启动时自动检测 LM Studio 是否运行 (`GET http://localhost:1234/v1/models`)
- **未启动处理**: 显示引导弹窗 ("检测到 LM Studio 未运行，请先启动服务")，任务队列保持 pending
- **自动重连**: 后台每 30 秒健康检查一次，服务恢复后自动恢复队列处理
- **端口变更**: 支持用户自定义 LM Studio 地址 (设置页配置)，保存到 `app_config` 表
- **降级策略**: LM Studio 离线期间，用户可继续浏览/管理已有图片，仅 AI 功能受限

### 2.3 模块 C: 智能去重 (Smart Deduplication)

#### 2.3.1 页面: 重复项管理 (DedupManager)
**入口**: 侧边栏 → "去重"

**UI 元素清单**:
| 元素名称 | 类型 | 交互逻辑 |
|---|---|---|
| 相似度阈值 | Slider | 70%-99%，默认 90% |
| 扫描按钮 | Button | 触发 pHash 相似度计算 |
| 对比视图 | Split Panel | 并排显示相似图片，高亮差异区域 |
| 保留选择 | Radio | 保留高分辨率/保留先导入 |
| 批量删除 | Button | 删除选中重复项，二次确认 |

**逻辑流程**:
1. 用户点击"扫描" → Rust 计算所有图片的 pHash
2. 对比 pHamming 距离，找出相似度 >= 阈值 的图片对
3. 显示重复项分组，用户逐组审核
4. 选择保留项 → 删除重复项 → 清理数据库

### 2.4 模块 D: 语义搜索 (Semantic Search) [Phase 2]

#### 2.4.1 页面: 搜索结果页
**路由**: `/search`

**UI 元素清单**:
| 元素名称 | 类型 | 交互逻辑 |
|---|---|---|
| 搜索框 | Input | 支持自然语言查询 |
| 结果网格 | Virtual Grid | 按相关性排序 |
| 筛选器组合 | Filter | 时间范围 + 分类 + 标签 |
| 关键词高亮 | Highlight | 匹配词条黄色背景 |

**搜索流程**:
1. 用户输入"找一张日落的海滩照片"
2. Rust 端 jieba-rs 分词 → ["日落", "海滩", "照片"]
3. 搜索 `search_index` 倒排索引表
4. JOIN `images` 表获取完整元数据
5. 返回图片列表 + 相关性分数 (匹配词条数)

## 3. 非功能需求 (Non-functional Requirements)

| 维度 | 要求 |
|------|------|
| **性能** | 5 万张图片列表滚动 60fps；应用启动 < 2 秒；缩略图生成不阻塞 UI |
| **安装包大小** | ≤ 15MB (纯 Rust 架构目标 ~10.5MB) |
| **内存占用** | 运行中 < 200MB (不含 LM Studio) |
| **兼容性** | Windows 10 1809+ (Build 17763) |
| **离线可用** | 核心浏览/管理功能无需网络 |
| **隐私保护** | 不采集/上传任何用户数据 |
| **无障碍** | MVP: 键盘导航 + 语义 HTML + 合理对比度 (Phase 2: 完整 WCAG 2.1 AA) |

## 4. 假设与约束

### 4.1 假设
- LM Studio 已安装且运行在 `localhost:1234`
- Qwen2.5-VL-7B-Instruct 模型已加载到 LM Studio
- 用户机器至少 16GB RAM (推荐 32GB)
- 磁盘空间充足 (应用 + 数据 < 500MB，不含图片)

### 4.2 约束
- 所有新代码必须位于 `e:\智能体项目优化\`，原项目仅参考
- 原项目 `D:\Personal\Desktop\arcane-codex-src\` 严禁修改
- 不依赖任何云服务 (除 LM Studio 本地服务)
- 仅 targeting Windows 10+
